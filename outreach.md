## 🎯 Who Is This For?

This project is designed for **students in AI, Data Science, or Computer Science** who are comfortable writing code, but have never taken a project beyond their local machine.

It is especially useful for those who:
- can build small applications or models  
- use GitHub regularly  
- are curious about cloud tools like AWS  
- want to understand how projects become accessible to real users  

At the beginning of this project, we were in a similar position ourselves — able to build things locally, but with limited experience in deploying them.

---

## ❓ Why Does This Exist?

In many technical courses, the workflow looks like this:

Build → Test → Submit → Move on

What is often missing is what happens after that — **how does your work actually reach users?**

Many students:
- never deploy their projects  
- never see real users interact with their work  
- never collect feedback beyond grading  

At the same time, most online resources for deployment:
- assume prior knowledge of cloud infrastructure  
- or introduce too many tools at once  

This project was created to provide a **simple and complete example** of taking a project from code to deployment, without unnecessary complexity.

---

## 🔍 What Makes This Difficult for Students?

When trying to learn deployment, students often face:

| Situation | Challenge |
|---|---|
| Watching tutorials | Hard to adapt to your own project |
| Reading AWS docs | Too technical for beginners |
| Using complex projects | Difficult to isolate core concepts |

As a result, deployment feels more confusing than it needs to be.

We wanted to create something that is:
- simple  
- understandable  
- and easy to replicate  

---

## 🔨 What We Built

To focus on learning rather than complexity, we built a **2048 game using Python (Flask)** and deployed it on AWS.

### Tech Stack

| Layer | Technology |
|---|---|
| Game Logic | Python |
| Backend | Flask |
| Frontend | HTML, CSS, JavaScript |
| Deployment | AWS EC2 |
| Documentation | GitHub Pages |
| Feedback | Google Forms |
| Database | SQLite (Leaderboard) |

### Features

- Playable 2048 game in the browser  
- Mobile swipe support  
- Dark mode toggle  
- Tile animations  
- Leaderboard with top scores  
- Score submission system  
- User feedback form  

The goal was not to build a complex application, but to create a **clear end-to-end workflow** that others can understand and reproduce.

---

## 📣 How the Project Was Shared

Due to time constraints, the project was shared within a **small but targeted group** rather than a large public audience.

This included:
- classmates  
- peers from the same academic program  
- friends with a technical background  

This approach allowed us to:
- get quick and relevant feedback  
- test usability in a controlled environment  
- iterate faster on improvements  

Participants were asked to:
1. Play the game  
2. Explore the repository and documentation  
3. Submit feedback through a short form  

---

## 📊 What We Learned from Users

Feedback was collected through a Google Form and analysed for common patterns.

### Most Requested Features

| # | Feature |
|---|---|
| 1 | Multiplayer mode |
| 2 | Sound effects |
| 3 | Tile animations |
| 4 | Undo move |
| 5 | Tutorial / onboarding |

Some features such as animations, mobile support, and leaderboard functionality were implemented after this feedback.

### Key Takeaways

- The application was easy to access and use  
- Mobile support was important for users *(and was improved)*  
- Feedback helped prioritise improvements  

Most importantly, this showed us that:

**Deployment is not the final step — it is where real learning begins.**

---

## 💡 What Makes This Project Different

Most student projects stop after development.

This project focused on completing the full cycle:

Build → Deploy → Share → Collect Feedback → Improve

By doing this, we were able to:
- understand how deployment works in practice  
- interact with real users  
- improve the project based on feedback  

Even with a simple application, this process made the project feel more complete and meaningful.
